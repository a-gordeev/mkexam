# mkexam — project requirements

## question requirements

each question must be derived from a single chapter, sections, paragraphs and other units of text greater than an isolated statement

for each question there should be 5 answers: 2 correct and 3 incorrect

the answers should differ in one-two words that try to fool the user and obfuscate correct answers

a question should address one clear idea, matter or fact

## quiz

on submit show all correct answers

provide skip and hide buttons above a question; submit - below

once the answers are submitted show the documentation exceprts that explain why correct answers are right and incorrect answers are wrong

title questions with the chapter, section, topic etc. stored on generation

## question generation

never invent/hallucinate the contents answers are generated from

every topic, chapter or section  should be covered

## PDF question generation

skip table of contents and other technical document information

mark each answer with the full path in the table of contents: chapter, section, page etc.

## video, youtube and other type of source generation

table of contents should be established internally as if it were a PDF document; then it should be worked through

once a table of contents established same rules as for PDF

use time marks instead of page numbers

skip jokes and other information that is unrelated to the subject

## packaging

recreate deployment package with every request for git commit
